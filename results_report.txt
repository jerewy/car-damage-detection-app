You are absolutely right. If the text isn't centered and the buttons are messy, it's usually because the CSS classes aren't interacting correctly with the `display: block` or `width` properties, especially in `print` mode.

Also, "Download PDF" is technically correct because clicking "Print" allows you to *Save as PDF*, but calling it **"Print / Save PDF"** is clearer for users.

Here is the **fixed and polished HTML template**. I have targeted the specific CSS issues causing the messy alignment.

### **The Fixes Applied:**

1.  **Vertical & Horizontal Centering:** Used `d-flex align-items-center justify-content-center` on the result boxes to force images and text to stay in the middle.
2.  **Clean "Print" Button:** Renamed to "Print / Save PDF" and styled it distinctively.
3.  **Print Layout Stability:** Added specific CSS to force the "Assessment Summary" table to stay aligned next to the images in the PDF view.
